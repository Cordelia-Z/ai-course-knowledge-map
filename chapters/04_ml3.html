二、层级知识结构

按“一级知识点 → 概念/公式/例子/重点/易错/考点”整理。

1. 泛化与数据划分

最终关心未见样本上的表现。

关键概念

  • 训练误差：模型在训练集上的错误。
  • 测试误差：模型在未参与训练数据上的错误。
  • 验证集：用于调参和选模型。

公式/规则

  • 本项不作为单独节点展开。

典型例子

  • 刷熟训练题不等于考试会做新题。

老师强调点

  • 训练误差低不代表测试好。

易错点

  • 易错：反复用测试集调参会污染测试集。

考试可能考法

  • 比较训练集、验证集、测试集。

2. 过拟合与欠拟合

模型复杂度和数据规模不匹配会导致泛化问题。

关键概念

  • 过拟合：训练好、测试差，模型记住噪声。
  • 欠拟合：训练和测试都差，模型能力不足或训练不足。
  • 模型复杂度：参数、特征、深度等影响拟合能力。
  • 数据规模：更多高质量数据通常能降低估计误差。

公式/规则

  • 本项不作为单独节点展开。

典型例子

  • 高次多项式拟合少量噪声点容易过拟合。

老师强调点

  • 泛化问题是 ML 的核心考点。

易错点

  • 易错：模型越复杂不一定越好。

考试可能考法

  • 根据训练/测试误差判断过拟合或欠拟合。

3. 误差分解

区分表示不够还是数据不够。

关键概念

  • 近似误差：假设类太弱，无法表示真实规律。
  • 估计误差：数据有限导致学到的参数偏离最优。

公式/规则

  • generalization gap = test error − train error

典型例子

  • 线性模型拟合强非线性数据是近似误差。

老师强调点

  • 要能判断错误来自哪里。

易错点

  • 易错：把所有测试差都归因于过拟合。

考试可能考法

  • 解释近似误差与估计误差。

4. 正则化与无监督学习

通过限制复杂度或寻找数据结构提高学习质量。

关键概念

  • 正则化：在目标函数中加入复杂度惩罚。
  • 超参数：训练前设定，如学习率、正则强度。
  • 无监督学习：无标签时发现结构、聚类或低维表示。

公式/规则

  • min_w Σ_i Loss_i(w)+λΩ(w)

典型例子

  • L2 正则惩罚过大的权重。

老师强调点

  • 正则化是处理过拟合的重要方法。

易错点

  • 易错：验证集调超参数，训练集学参数。

考试可能考法

  • 解释正则化目标。
